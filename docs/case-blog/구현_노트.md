# 진료케이스 블로그 3단계 — 구현 노트

스키마·1·2·3단계 프롬프트 문서대로 구현 완료(2026-06-07).

## 흐름
차트 상세 "진료케이스 작성" → 모달 3단계 위저드
1. **인과 흐름**(causalFlow) — thinking 켬(thinkingBudget 4096, maxOutputTokens 16384)
2. **섹션 아웃라인**(outline) — thinking 0, maxOutputTokens 8192. 입력: 검수된 causalFlow
3. **블로그 글**(blogPost) — thinking 0, maxOutputTokens 12288. 입력: 검수된 outline + 병원명/환자명 + 좋은예시(선택). blog_post로 저장(진료케이스 메뉴에서 열람)

각 단계 산출물은 DB에 저장(`generated_run_content`): `blog_causal`, `blog_outline`, `blog_post`.
모달 다시 열면 저장된 단계까지 복원.

## 코드 위치
- chart-api: `app/api/content/generate/route.ts` (contentType: blog_causal / blog_outline / blog_post), SYSTEM 프롬프트 상수 SYS_CAUSAL/SYS_OUTLINE/SYS_BLOGPOST
- chart-api: `lib/chart-app/gemini.ts` — geminiGenerateText에 systemInstruction·thinkingBudget 옵션 추가
- admin-web: `components/admin-case-blog-modal.tsx` (3단계 위저드)
- admin-web: `app/api/admin/health-report/content/route.ts` PATCH 화이트리스트(blog_causal/outline/post)
